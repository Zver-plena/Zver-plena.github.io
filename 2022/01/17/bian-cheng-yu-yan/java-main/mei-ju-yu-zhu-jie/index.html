2021.12.14-2021.12.20

1. 枚举类

1.1.枚举类的理解

  1. 类的对象是有限的,确定的,我们称此类为枚举类

    1. 举例如下:

      • 星期:

      • 季节:

      • 线程状态:创建,就绪,运行,堵塞,死亡.

  2. 当需要定义一组常量的时,强烈建议使用枚举类

  3. 如果枚举类中只有一个对象,则相当于单例模式的实现方式

1.2.枚举类的创建

1.2.1.自定义创建枚举类

  1. 构造器私有化
  2. 类中创建public static 对象
代码实现
class Week {
    private String weekDecription;

    /**
     *      1.构造器私有化
     */
    private Week() {

    }
    private Week(String weekDecription) {
        this.weekDecription = weekDecription;
    }

    /**
     *      2.在类中创建对象
     */
    public static final Week MON = new Week("MON");
    public static final Week TUS = new Week("TUS");
    public static final Week WED = new Week("WED");
    public static final Week THU = new Week("THU");
    public static final Week FRI = new Week("FRI");
    public static final Week SAT = new Week("SAT");
    public static final Week SUN = new Week("SUN");

    public String getWeekDecription() {
        return this.weekDecription;
    }

}

2.注解(Annotation)

2.1.注释的理解

框架 = 注解 + 反射 + 设计模式

Annotation就是代码里面的特殊标记,这些标记可以在编译,类加载,运行时被读取,并执行相应的处理. 可以在不改变原有逻辑的情况下,在源文件添加一些补充信息

2.2.Annotation的使用示例

  1. @author:开发者

  2. @version:该模块的版本

  3. @see:参考转向,相关主题

  4. @since : 从那个版本开始添加的;

  5. @param : 对方法中某参数的说明,如果没有参数不用写

  6. @return : 对方返回值的说明,如果方法的返回值类型是void就不能写

  7. @exception : 对可能抛出的异常进行说明,如果方法没有用throws显示抛出的异常就不用写

    在编译时进行格式检查

    1. @Override : 限定重写父类方法,该注解只能用于方法

    2. @Deprecated : 用于表示所修饰的元素(类,方法等)已过时,通常是因为所修饰的结构危险或存在更好的选择

    3. @SuppressWarnings : 抑制编译器警告

跟踪代码的依赖性,实现替代配置文件功能

2.3.如何自定义注解

  • 定义新的Annotation类型需要使用@interface关键字.

  • 自定义注解自动继承java.lang.annotation的Annotation接口

  • 内部定义成员,通常使用value表示

  • 可以指定成员的默认值,使用default定义

  • 如果自定义注解没有成员,表明是一个标识作用.

如果注释有成员,那么就需要指明成员的值,如果有默认值可以不写.

代码实现
public @interface MyAnnotation {
    String value()default "100";//default 默认值
}

2.4.jdk中4个基本的注解的使用

JDK的元注解用于修饰其他Annotation定义

元注解

JDK5.0提供了4个标准的meta-annotation类型,分别是

Retention

@Retention用于指定该Annotation的生命周期,@Retention包含一个RetentionPolicy类型的成员变量.

包—–import java.lang.annotation.Retention

使用Retention必须为该value成员变量指定值,RetentionPolicy其有三种状态(枚举类)

  1. RetentionPolicy.SOURCE : 在源文件中有效

    1. RetentionPolicy.CLASS — 默认状态 : 在编译时保留在CLASS文件里面,但是在运行java程序时,JVM不会保留注解

  2. RetentionPolicy.RUNTIME : 在编译和运行的时候都保留注解

Target

@Target用于指定被修饰的Annotation能用于修饰那些程序元素

包—–import static java.lang.annotation.ElementType.*

@Target 也包含一个名为value的成员变量

  • CONSTRUCTOR — 构造器
  • FIELD — 域
  • LOCAL_VARIABLE — 局部变量
  • METHOD — 方法
  • PACKAGE — 包
  • PARAMETER — 参数
  • TYPE — 类,接口或enum声明
  • TYPE_USE:表明该注解能写在使用类型的任何语句中—jdk8
  • TYPE_PARAMETER : 表示该注解能写在类型变量的声明语句中(如:泛型声明)—jdk8
Documented —- 出现频率较低

用于指定被该元Annotation修饰的Annotation类,将被javadoc工具提取成文档
表示所修饰的注解在被javadoc解析时,保留下来

默认情况下,javadoc是不包含注解的

定义为Documented的注解必须设置Retention的值为RUNTIME

Inherited — 出现频率较低

Inherited被他所修饰的 Annotation 将具有继承性

2.5.利用反射获取注解信息

2.6.jdk8新特性:可重复注解

JDK8之前实现重复注解

  1. 创建一个MyAnnotaion的数组value的注解
代码实现
public @interface MyAnnotations {
    MyAnnotation[] value();
}
-------------------------------
public @interface MyAnnotation {
    String value()default "100";
}
------------------------------
@MyAnnotations({@MyAnnotation(value="23"),@MyAnnotation (value="11")})

JDK8实现

思路
  1. 在MyAnnotation上声明@Repeatable,成员值为MyAnnotations.class
  2. MyAnnotation的的Target和Retension要与MyAnnotations相同.
代码实现
import static java.lang.annotation.ElementType.*;
//2.两个注释内容要同步
@Retention(RetentionPolicy.RUNTIME)
@Target({TYPE,METHOD})
// 1. 实现@Repeatable
@Repeatable(MyAnnotations.class)
public @interface MyAnnotation {
    String value()default "100";
}
--------------------------------------------------------------------
@Retention(RetentionPolicy.RUNTIME)
@Target({TYPE,METHOD})
public @interface MyAnnotations {
    MyAnnotation[] value();
}
--------------------------------------------------------------------
    @MyAnnotation(value="23")
    @MyAnnotation(value="11")

2.7.jdk8新特性:类型注解

支持注解泛型

@Target里面有个参数TYPE_PARAMETER.

TYPE_PARAMETER : 表示该注解能写在类型变量的声明语句中(如:泛型声明)

代码
@Retention(RetentionPolicy.RUNTIME)
@Target({TYPE,METHOD,TYPE_PARAMETER})
public @interface MyAnnotation {
    String value()default "100";
}

支持注解异常

@Target里面有个参数TYPE_USE.

TYPE_USE:表明该注解能写在使用类型的任何语句中

代码
@Retention(RetentionPolicy.RUNTIME)
@Target({TYPE,METHOD,TYPE_USE})
@Repeatable(MyAnnotations.class)
public @interface MyAnnotation {
    String value()default "100";
}
