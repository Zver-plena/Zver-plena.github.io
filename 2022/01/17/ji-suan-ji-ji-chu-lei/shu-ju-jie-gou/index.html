什么是数据结构

数据结构的定义

数据结构是研讨数据的逻辑结构和存储结构,以及他们之间的相互关系,并对与这种结构定义相应的操作,设计出相应的算法.

数据结构的逻辑结构

数据的逻辑结构是指数据的各数据元素之间的逻辑关系

  • 数据的逻辑结构分为两大类

    1. 线性结构

    2. 非线形结构

      • 树形结构

数据结构的物理结构

数据的物理结构是指数据在计算机内的实际存储形式。

基本概念和术语

数据

  • 数值型数据
  • 非数值型数据

数据元素

  • 数据的基本单位
    • 一个数据元素可由若干个数据项组成
    • 数据项是数据不可分割的最小单位

数据对象

  • 数据的子集
    • 具有相同性质的数据元素集合

数据结构

  • 数据结构是相互间存在一种或多种特定关系的数据元素集合
    • 结构:数据元素相互之间的关系
    • 特定关系
    • 数据结构按逻辑结构可以分为两大类
      • 线性结构
      • 非线性结构

数据结构的形式定义

  • 二元组: DS=(D,S)
    • D : 数据元素的有限集合
    • S : 定义在D上的关系的有限集合

数据的物理结构(存储结构)

  • 数据元素之间关系的表示
    • 顺序映像:
      • 借助元素在存储器的相对位置表示数据元素之间的逻辑关系.对应于顺序存储结构
    • 非顺序映像:
      • 利用指示元素存储地址的指针表示数据元素间的逻辑关系对应于
        • 链式存储结构
        • 索引树
        • 散列表

算法

算法原地工作

算法原地工作: 是指算法所需辅助空间是常量，即O(1)。

线性表

线性表的定义与特点

线性表的定义

线性表是n个数据元素的有限序列

线性表的特性

  • 元素间关系是线性的
  • 同一线性表中的元素必定具有相同特性

线性表的顺序表示和实现

顺序表的表示

线性表的顺序表示: 指的是一组地址连续的存储单元依次存储线性表的数据元素.

这种表示又称为顺序存储结构或顺序映像,通常称这种存储结构的线性表为顺序表

  • 顺序存储定义:
    • 把逻辑上相邻的数据元素存储在物理上相邻的存储单元中的存储结构
    • 简言之,逻辑上相邻,物理上也相邻
  • 顺序存储方法:
    • 用一组地址连续的存储单元依次存储线性表的元素，可通过数组来实现

链式存储的术语

结点

数据元素的存储映像,由数据域和指针域组成.

链表

n个结点由指针链组成一个链表,他是线性表的链式存储映像,称为线性表的链式存储结构

单链表

结点只有一个指针域的链表,称为单链表或线形链表.

双链表

有两个指针域的链表

多链表

有多个指针域的链表

循环链表

循环链表是首尾相接的链表

头指针

头指针是指向链表中第一个结点的指针

单链表可由一个头指针唯一确定

头结点

头结点是在链表的首元结点之前附设的一个结点,数据域内只放空表标志和表长等信息.

知识点
  1. 在链表中设置头结点有什么好处
  1. 头结点是在链表的首元结点之前附设的一个结点,作用是对链表进行操作时,可以对空表,非空表的情况以及对首元结点进行统一处理,编程更方便
  1. 如何表示空表
  1. 无头结点时,当头指针的值为空时表示空表.
  2. 有头结点时,当头结点的指针域为空时表示空表

.二叉树

二叉树

二叉树的定义

  1. 有且仅有一个根结点
  2. 每个结点至多有两棵子树(即二叉树中不存在度大于 2 的点).
  3. 二叉树的子树有左右之分,且次序不能随便颠倒.

二叉树的存储结构

二叉树的性质

  1. 在二叉树的第i层上至多有 $2^{i-1}$ 个结点$(i>=1)$

  2. 深度为k的二叉树至多有 $2^{k}-1$ 个结点$(k>=1)$

  3. 对于任何一颗二叉树T,如果其终端结点数为n0,度为2的结点数为n2,则会有n0 = n2 + 1.

  4. 设n1为二叉树中T中度为 1 的结点树,因为二叉树中的所有结点的度均小于或者等于 2 ,所以其结点总数为n = n0 + n1 + n2

    —设B为分支总数,那么有n=B+1,由于这些分支是由度为1或2的结点射出的,

    所有又有B = n1 + 2n2

    —结合n = B + 1和B = n1 + 2n2的结论

​ 可以得到n = n1 + 2n2 + 1

​ 整合又可以得n0 = n2 + 1.

完全二叉树

基本概念

完全二叉树 : 是由满二叉树而引出来的，若设二叉树的深度为h,有n个结点，除第 h 层外，其它各层 (1～h-1) 的结点数都达到最大个数(即1~h-1层为一个满二叉树)，第 h 层所有的结点都连续集中在最左边，这就是完全二叉树

完全二叉树的性质

  1. 具有n个结点的完全二叉树的深度为$log_2n+1$(向下取整)

    由二叉树的性质以及完全二叉树的定义可得

    $2^{k-1}-1<n\le2^{k}-1$

    或者得

    $2^{k-1}\le n < 2^k$

    取对数得到

    $k-1\le lox_2n < k$

    因为k是整数,所以

    $k=log_2n+1$

构造

知识点

  1. 问题一: 一个具有n个节点的完全二叉树,其叶子节点的个数为多少？

$n=n_0+n_1+n_2$

二叉树性质有

$n_0=n_2+1$

所以得到

$n_0=(n+1-n_1)/2$

因为由完全二叉树可知,$n_1=1或n_1=0$

所以得到

  1. $n_0=(n+1)/2$ .即n为奇数时
  2. $n_0=n/2$ .即n为偶数时

平衡二叉树

平衡二叉树的定义

  • 对于每一个节点而言，左右子树的深度之差<=1

如何判断一个二叉树为平衡二叉树

1.分别求左右子树的深度，再进行判断、递归。（此方法会遍历一个节点多次，效率不高。）


typedef struct BinaryTreeNode {
    struct BinaryTreeNode *left;
    struct BinaryTreeNode *right;
    int data;
};
//1.分别求左右子树的深度，再进行判断、递归。（此方法会遍历一个节点多次，效率不高。）
int TreeDepth(BinaryTreeNode *pRoot){
    if(pRoot==NULL){
        return 0;
    }
    return 1+TreeDepth(pRoot->left)+TreeDepth(pRoot->right);
}
bool IsBalanced_solve(BinaryTreeNode *pRoot){
    if(pRoot==NULL) return true;
    int left = TreeDepth(pRoot->left);
    int right = TreeDepth(pRoot->right);
    if(abs(right-left)>1) return false;
    return  IsBalanced_solve(pRoot->left) && IsBalanced_solve(pRoot->right);
}

2.用后序遍历的方式遍历二叉树的每一个节点，在遍历到一个节点之前我们就已经遍历了它的左右子树。只要在遍历每个节点的时候记录深度，就可以一边遍历一边判断每个节点是不是平衡的。（更高效的方法）

bool IsBanlanced(BinaryTreeNode *pRoot , int * depth){
        if(pRoot==NULL){
            *depth=0;
            return true;
        }
        int left=0,right=0;
        if(IsBanlanced(pRoot->left,&left)&&IsBanlanced(pRoot->right,&right)){

5.图

图的基本概念

  • 图(G) : 是由两个集合V和E组成,标记为G=(V,E);

    • V(G)表示图G的顶点集合

      E(G)表示图G的边集合

  • 有向图: 对于图(G),如果E(G)为有向边的集合,那么G就是有向图

6.1.2.折半查找

思路

二分查找每次都是取区间的中点来比较mid=(l+r)/2=(l+r)>>1

递归左右区间

代码实现
/*
 * @Autor: Zver
 * @Date: 2021-12-11 16:14:57
 * @LastEditTime: 2021-12-11 22:58:09
 */
#include <iostream>
using namespace std;
const int N = 10010;
typedef struct {
    int value[N];
    int numberCount;
} Array;
Array arr;
//快速排序
void qsort(Array& arr, int l, int r)
{
    if (l >= r)
        return;
    int mid = arr.value[(l + r) >> 1];
    int ls = l - 1, rs = r + 1;
    while (rs > ls) {
        do
            ls++;
        while (arr.value[ls] < mid);
        do
            rs--;
        while (arr.value[rs] > mid);
        if (ls < rs)
            swap(arr.value[ls], arr.value[rs]);
    }
    qsort(arr, l, rs);
    qsort(arr, rs + 1, r);
}
//二分查找
int binaryFind(Array& arr, int number, int l, int r)
{
    while (r > l) {
        int mid = (l + r + 1) >> 1;
        if (arr.value[mid] > number)
            r = mid - 1;
        else
            l = mid;
    }
    return l;
}
int main()

{
    //读入数据
    cin >> arr.numberCount;
    for (int i = 0; i < arr.numberCount; i++) {
        cin >> arr.value[i];
    }
    //读入查找
    int count, findNumber;
    cin >> count >> findNumber;
    for (int i = 0; i < count; i++) {
        cin >> findNumber;
        //实现查找操作
        arr.value[arr.numberCount] = findNumber;
        int index = binaryFind(arr, findNumber, 0, arr.numberCount);
        if (index == -1) {
            cout << "查找失败,不存在这个数" << endl;
        } else {
            cout << "查找成功,下标:";
            cout << index << endl;
        }
    }

    return 0;
}

6.2.1.二叉排序树

二叉排序树的定义
  1. 如果左子树不空,则左子树的所有结点的值均小于他的根结点的值
  2. 如果右子树不空,则右子树的所有结点的值均小于他的根结点的值,
  3. 他的左右子树已分别为二叉排序树.
二叉排序树性能

时间复杂度平均而言和$O(log_2n)$同数量级.

性质

对一个结点而言

  1. 左儿子小于父结点
  2. 右儿子大于父结点
代码实现
/*

 * @LastEditTime: 2021-12-11 18:27:44

#include <iostream>
using namespace std;

typedef struct BSTNode {
    int data;
    struct BSTNode *lchild, *rchild;
} BSTNode, *BSTree;
BSTree tree;
//插入操作
void insert(BSTree& T, int number)
{
    if (T == NULL) {
        BSTree s = new BSTNode;
        s->data = number;
        s->lchild = s->rchild = NULL;
        T = s;
    } else {
        if (number > T->data)
            insert(T->rchild, number);
        else if (number < T->data)
            insert(T->lchild, number);
        else
            return;
    }
    return;
}
//中序遍历
void midList(BSTree& T)
{
    if (T != NULL) {
        cout << T->data << " ";
        midList(T->lchild);
        midList(T->rchild);
    }
    return;
}
//创建操作
void build(BSTree& T)
{
    T = NULL;
    int number;
    int count = 0;
    cin >> count;
    for (int i = 0; i < count; i++) {
        cin >> number;
        insert(T, number);
    }
}
//查找操作
bool find(BSTree& T, int number)
{
    if (T) {
        if (T->data == number)
            return true;
        else if (T->data > number)
            return find(T->lchild, number);
        else if (T->data < number) {
            return find(T->rchild, number);
        }
    }
    return false;
}
int main()
{
    build(tree);
    midList(tree);
    cout << endl;
    int count, number;
    cin >> count;
    for (int i = 0; i < count; i++) {
        cin >> number;
        if (find(tree, number)) {
            cout << "查找成功" << endl;
        } else
            cout << "查找失败" << endl;
    }
    return 0;
}

散列函数的构造方法

构造“好”的散列函数遵循原则
  1. 函数计算要简单,每一个关键词只能有一个散列地址与之对应;
  2. 函数的值域在表长的范围内,计算出的散列地址应该分布均匀,减少冲突.

    $H(key)=key \mod p$

    这个方法的关键是选取适当的p,一般情况下p应该选取为小于表长的最大质数.

散列表的查找

算法步骤
  1. 给定待查找的关键词key,根据散列函数转换成散列下标$H_0=H(key)$
  2. 如果$Hash[H_0]$为空,那么所查元素不存在
  3. 如果$Hash[H_0]中的元素的关键词为key$,那么查找成功
  4. 否则重复下述过程解决冲突
    • 按照处理冲突的方法,计算下一个散列下标$H_i$;
    • 如果$Hash[H_i]为空$,那么所查的元素不存在
    • 如果$Hash[H_i]==ke y$,则查找成功

  2. 可用于链式结构
  3. 移动记录较多,算法平均时间性能比直接插入排序差,当初始记录无序,n较大时,此算法不宜采用

typedef struct node {
    int value[MAXSIZE];
    int length;
} SqList;

void BbSort(SqList a)
{
    int length = a.length;
    bool flag = 1;
    while (length > 0 && flag) {
        flag = 0;
        for (int i = 0; i < length - 1; i++) {
            if (a.value[i] > a.value[i + 1]) {
                flag = 1;
                swap(a.value[i], a.value[i + 1]);
            }
        }
        --length;
    }
}

快速排序

  • 不稳定排序
算法思想

在冒牌排序过程中,只对相邻的两个记录进行比较,因此每次交换两个相邻记录只能消除一个逆序,如果能通过两个不相邻的记录的一次交换,消除多个逆序,则会大大加快排序的速度.

快速排序的一次交换可以消除多个逆序.

代码实现
void qsort(SqList arr, int l, int r)

        while (arr.value[ls] < mid);
        do
            rs--;
        while (arr.value[rs] > mid);
        if (ls < rs)
            swap(arr.value[ls], arr.value[rs]);

}

堆排序

算法思路

首先把无序序列建成堆(大根堆或者小根堆)

反复交换堆顶元素和最后一个元素,在调整为堆(大根堆或者小根堆)

保证每次都是一个当前最大的数有序,类似于冒泡排序

算法复杂度

堆排序不稳定,但是最坏的情况下,其时间复杂度也为$O(nlog_2n)$

平均性能接近最坏性能.

仅需要一个记录大小供交换用的辅助存储空间,所以空间复杂度为O((1).

算法特点
  1. 是不稳定排序
  2. 只能用于顺序结构,不能用于链式结构
  3. 建初始堆所需要的比较次数较多,当记录数较少时不宜采用,当记录数较多,相较于快速排序的最差情况的$O(n^2)$,其最差情况也是$O(nlog_2n)$较为高校.
