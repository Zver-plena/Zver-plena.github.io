v-bind

缩写

<!-- 完整语法 -->
<a v-bind:href="url">...</a>

<!-- 缩写 -->
<a :href="url">...</a>

<!-- 动态参数的缩写 (2.6.0+) -->
<a :[key]="url"> ... </a>

//根据seen的真假来操作元素

v-show和v-if的用法大致一样

v-if vs v-show

v-if

  • v-if 会确保在切换的过程中，条件块内的事件监听器和子组件被销毁和重建
  • v-if如果为假什么都不会处理，直到为真才开始渲染条件块

v-show

  • 不管初始条件是什么，元素总是会被渲染，只是简单的基于css的切换

v-if有更高的切换开销，v-show有更高的渲染开销

  • 频繁切换 v-show
  • 不频繁 v-if

事件驱动

v-on：对应的vue对象里面的函数名

<script>
    var app = new Vue({
	"el":"#app",
	"data":{

Data 和 method

method

  • Vue 自动为 methods 绑定 this，以便于它始终指向组件实例。这将确保方法在用作事件监听或回调时保持正确的 this 指向。
  • 在定义 methods 时应避免使用箭头函数，因为这会阻止 Vue 绑定恰当的 this 指向。
const app = Vue.createApp({
  data() {
    return { count: 4 }
  },
  methods: {
    increment() {
      // `this` 指向该组件实例
      this.count++
    }
  }
})

防抖和节流

?没了解过

Vue 没有内置支持防抖和节流，但可以使用 Lodash 等库来实现。

在method中应用防抖

<script>
  Vue.createApp({
    methods: {
      // 用 Lodash 的防抖函数
      click: _.debounce(function() {
        // ... 响应点击 ...
      }, 500)
    }
  }).mount('#app')
</script>

注意事项

这种方法对于可复用组件有潜在的问题，因为它们都共享相同的防抖函数。为了使组件实例彼此独立，可以在生命周期钩子的 created 里添加该防抖函数:

app.component('save-button', {
  created() {
    // 使用 Lodash 实现防抖
    this.debouncedClick = _.debounce(this.click, 500)
  },
  unmounted() {
    // 移除组件时，取消定时器
    this.debouncedClick.cancel()
  },
  methods: {
    click() {
      // ... 响应点击 ...
    }
  },
  template: `
    <button @click="debouncedClick">
      Save
    </button>
  `
})
