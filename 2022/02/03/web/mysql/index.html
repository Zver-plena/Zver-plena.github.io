Mysql菜鸟教程

Mysql概述

SQL(发音为字母S-Q-L或sequel)是结构化查询语言(Structured Query Language)的缩写

数据库分类

关系型型数据库：

  • MySQL, Oracle, Sql Server, DB2, SQLite
  • 通过表和表之间的，行和列之间的关系进行数据的存储
  • 例如 学员信息表

非关系型数据库：

  • Redis MongDB
  • 通过存储对象自身的属性决定

DBMS（data base massage system)

  • 数据库的管理软件，科学有效的管理我的数据，获取和维护数据；
  • MySQL 就是关系型DBMS，运用SQL语言访问数据库的标准化语言.

MySQL 使用

MySQL的命令对大小写都一样

MySQL以分号结尾

进入mysql

	mysql -u root -p
# -u ： 用户名
# -p : 密码	,注意-p和密码之间不要有空格

MAC下起别名

# 加载.zshrc里面每次启动终端就会加载
alias mysql=/usr/local/mysql/bin/mysql
alias mysqladmin=/usr/local/mysql/bin/mysqladmin

创建数据库

create database 数据库名;

使用 mysqladmin 创建数据库

使用普通用户，你可能需要特定的权限来创建或者删除 MySQL 数据库。

所以我们这边使用root用户登录，root用户拥有最高权限，可以使用 mysql mysqladmin 命令来创建数据库。

以下命令简单的演示了创建数据库的过程，数据名为 RUNOOB:

mysqladmin -u root -p create RUNOOB
# 以上命令执行成功后会创建 MySQL 数据库 RUNOOB。

数据库语言

DDL(data definition language） : 数据定义语言

DML(data manage language) : 数据操作语言

DQL(data query language) ： 数据查询语言

DCL(data control language) : 数据控制语言

数据库的列类型

数字

  1. tinyint: 1字节

  2. smallint: 2字节

  3. mediumint: 3字节

  4. int: 4字节

  5. bigint： 8字节

  6. double： 浮点数 8字节

  7. float ： 浮点数 4字节

  8. decimal ： 字符串形式的浮点数。 金融计算的时候用

字符串

  1. char 固定大小0~255
  2. varchar 可变字符串 0~65535
  3. tinytext 微型文本 2^8-1
  4. text 文本 2^16-1

时间日期

  1. data : YYYY-MM-DD 日期格式
  2. Time : HH ： mm : ss 时间格式
  3. datatime : YYYY-MM-DD HH : mm : ss
  4. timestamp : 时间戳 1970.1.1到现在的毫秒数

数据库的字段类型*

Unsigned

  • 无符号的整数

  • 声明了该列不能为负数

Zerofill

  • 0填充

  • 不足的位数用0进行填充

Auto increment(自增)

  • 自动在上一条记录的基础上+1
  • 通常用来设置唯一的主键

Decimals

  • 默认值

Not NUll

  • 当前的属性不能为空，如果为空会报错

primary key

  • 主键
  • 可以用来设置自增

对Mysql数据库的操作

创建数据库


create database if not exists 数据库名;
表示如果不存在这个数据库创建

修改数据库

alter datebase 数据库名
	#设置字符集
	default character set utf8

删除数据库

drop database if exists 数据库名;
# 删除所有表包括其中的数据

显示当前拥有的数据库

show databases;

转换数据库

use 数据库名;

对Mysql表的操作

创建表

# 1.创建现成的表语法
这两个必填，其他可选
CREATE TABLE IF NOT EXISTS (表名)(
	`字段名` 列类型（长度） [] [] [] [],
    `字段名` 列类型（长度）[] [] [] []
)[表类型][字符集]
-------------------------------------------------
# 1.创建现成的表实例
CREATE TABLE IF NOT EXISTS `Student`(
	`id` int(4) NOT NULL auto_increment COMMENT '学号', 
	`name` varchar(30),
	PRIMARY KEY(`id`) -- 设置主键
)
创建一个id列属性，列类型为int，长度为4，不能为空值（NOT NULL），自增（auto_increment）注释（comment）
创建了一个name列属性，列类型为varchar，长度为30，其他配置全为NULL

# 2.复制表
	#2.1 复制表结构,包含主键外键，不包含内容
	create table 表名 like 表名; 
	#2.2 复制表表结构,包含内容，不包含主键外键
	create table 表名 as select * from 表名

修改表

  • add
  • modify
  • change 列名 原列名
  • drop column 列名
  • rename to 新表名
-- ALTER(变更) ： 添加类型
-- 添加列类型
ALTER table  表名 ADD sex varchar(30) COMMENT 注释;
-- 删除列类型
alter table 表名 drop column 列名;
-- 修改列名
alter table 列名 rename to 列名
-- 修改列类型
ALTER table 表名 MODIFY column 列名 enum('男','女') not null default '男';

删除表

drop table if exists 表名;
-- 该命令会将表的描述，完整性约束，索引以及相关权限一并删除

显示当前数据库下所有的表

show tables;

对Mysql表内记录的操作

Mysql表的插入记录

插入新的记录
-- INSERT(插入) : 添加内容
-- 没有指明列类型要按照对应列类型顺序，依次填入
INSERT INTO `Student` VALUES (null,'小白','男');

-- insert指定一个列类型多个value，value之间用逗号隔开
INSERT INTO `Student` (`sex`) VALUES ('man'),('value');

-- 指定多个列类型
INSERT INTO `Student` (`name`,`sex`) VALUES ('xiaoyu','man'),('xiaola', 'woman');
从已有表中插入新纪录
insert into 表名 select [列名] from 表名;

Mysql表的修改记录

替换旧记录
当插入一个记录，这个记录的主键在表中已经存在可以使用 replace 替代 insert 此时旧记录会被覆盖
replace into `表名` values('','','');
修改单个表
-- 修改 table 为 `Student` 的`id`=1 的`name`
UPDATE `Student` set `name`= 'xiaoll' where `id`=1;

-- 不指定条件，会改动所有表； 这个时候你的数据库就废了
UPDATE `Student` SET `name`= "xx";

-- 修改多个属性之间要用逗号隔开
UPDATE `Student` SET  `name`='xiaoll', `sex` = 'woman' WHERE `sex`='man';

-- 语法
UPDATA 表名 SET 列名 = 值 , 列名 = 值  WHERE 条件;
修改多个表
update 表名1,表名2  set 表名1.列名=值,表名2.列名=值 where 条件；

Mysql表的删除记录

删除满足条件的行
-- 语法
delete from `表名` where 条件;

-- 例子
delete from `Student` WHERE `id` < 5;
从多个表中删除行
-- 语法
delete 表名1,表名2 from 表名1.列名,表名2.列名 where 条件;
消除表数据
-- 全部删完
delete from `Student`;

-- 删除一个数据库表，表的结构和索引约束不会变
truncate `Student`;



/*
	 使用delete 和 truncate的区别
	 delete 删除完 不会影响自增
	 truncate 删除完 自增会清零.
*/


`DELETE删除问题`，重启数据库，现象
- InnoDB 	自增列会从1 开始 （存在内存当中，断电即失）
- MyISAM   继续从上一个自增量开始（存在文件当中不会丢失）

SQL通配符过滤

  • 链接教程

  • 在 SQL 中，通配符与 like 操作符一起使用实现模糊查询。

通配符描述
%替代0个或多个字符
_替代一个字符
[charlist]字符列中的任何单一字符
[^charlist]或者[!charlist]不再字符列中的任何单一字符

使用 SQL % 通配符

-- 1.SQL 语句选取 url 以字母 "https" 开始的所有网站
SELECT * FROM Websites WHERE url LIKE 'https%';
-- 2.SQL 语句选取 url 包含模式 "oo" 的所有网站
SELECT * FROM Websites WHERE url LIKE '%oo%';

使用 SQL _ 通配符

-- 3.SQL 语句选取 name 以一个任意字符开始，然后是 "oogle" 的所有客户
SELECT * FROM Websites WHERE name LIKE '_oogle';
-- 4.SQL 语句选取 name 以 "G" 开始，然后是一个任意字符，然后是 "o"，然后是一个任意字符，然后是 "le" 的所有网站：
SELECT * FROM Websites WHERE name LIKE 'G_o_le';

SQl[charlist]

MySQL 中使用 REGEXP 或 NOT REGEXP 运算符 (或 RLIKE 和 NOT RLIKE) 来操作正则表达式。

-- 1.SQL 语句选取 name 以 "G"、"F" 或 "s" 开始的所有网站
SELECT * FROM Websites WHERE name REGEXP '^[GFs]';
-- 2.SQL 语句选取 name 以 A 到 H 字母开头的网站：
SELECT * FROM Websites WHERE name REGEXP '^[A-H]';
-- 3.SQL 语句选取 name 不以 A 到 H 字母开头的网站：
SELECT * FROM Websites WHERE name REGEXP '^[!A-H]';

通配符总结

正如所见，MySQL的通配符很有用。但这种功能是有代价的:通配 符搜索的处理一般要比前面讨论的其他搜索所花时间更长。这里给出一 些使用通配符要记住的技巧。

  • 不要过度使用通配符。如果其他操作符能达到相同的目的，应该 使用其他操作符。

  • 在确实需要使用通配符时，除非绝对有必要，否则不要把它们用 在搜索模式的开始处。把通配符置于搜索模式的开始处，搜索起 来是最慢的
