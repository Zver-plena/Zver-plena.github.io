Django理解

什么是Django?

  1. Django是一个开放源代码的Web应用框架，由Python写成。采用了MTV的框架模式，即模型M，视图V和模版T

自带管理后台，只需几行简单代码的设置，就可以让目标网站拥有一个强大的管理后台，轻松对内容进行增加、删除、修改与查找，并且能很方便地定制搜索、过滤等操作。

灵活的路由系统，可以定义优雅的访问地址，按需定义，毫无障碍。

强大的数据库 ORM，拥有强大的数据库操作接口(QuerySet API)，可以轻松执行原生 SQL。

易用的模板系统，自带强大、易扩展的模板系统。

MVC模式

将一个应用程序分解成三个组成部分:

  1. model–(模型)—M–管理应用程序的状态(通常存储到数据库中)
  2. view–(视图)—V–负责把数据格式化后呈现给用户
  3. controller–(控制器)–C–接受外部用户的操作,根据操作访问model获取数据,调用view呈现给用户,controller将model和view隔离并成为两者的纽带

django的运行模式

其实django也是一个MVC模式,但是在django中,控制器接受用户输入的部分由框架执行处理,所以django更关注的是

  1. model–(模型)–M,即数据存储层,这里处理与数据有关所有事件;

  2. template–(模板)-T,即表现层,该层处理与表现相关的决定;

  3. view–(视图)-V,即业务逻辑层,该层包含存储model以及调取template的相关逻辑,也可以说是template和model之间的桥梁

Django的作用?

  1. Django可用于创建动态的高安全性web应用程序。也就是说，通常由数据库支持的软件，包括某种通过浏览器操作的交互性。框架为这种软件的开发提供了一种结构和常用的方法。

class Player extends GameObject{


    constructor(playground, x, y, radius, color, move_speed, is_me){
        super();
        this.playground=playground;
        this.x=x;
        this.y=y;
        this.ctx=this.playground.gameMap.ctx;//Map canvas
        this.radius = radius;//Radius of the circle
        this.color = color;//player color
        this.move_speed = move_speed;//Player movement speed
        this.is_me = is_me;// Judge if it's yourself
        this.eps = 0.1; //Radius judgment deviation
        this.vx=1;//x increment
        this.vy=1;//y increment
        
    }
    start(){
    }
    update(){
        this.x=this.x+this.vx;
        this.y=this.y+this.vy;
        this.render();
    }
    render(){//Render the screen
        this.ctx.beginPath();
        this.ctx.arc(this.x,this.y,this.radius,0,Math.PI*2,false);
       this. ctx.fillStyle = this.color;
       this. ctx.fill();
    }

}
